## Directory Purposes

**`src/app/api/db/`:**
- Purpose: Core CRUD endpoints for all data entities
- Contains: `articles/` (CRUD + views + sidebar), `settings/`, `comments/`, `newsletter/`, `access-logs/`, `activity-logs/`, `view-logs/`, `distribute-logs/`, `auto-news-settings/`, `auto-press-settings/`, `api-keys/`
- Key files: `articles/route.ts` (main articles CRUD handler)

**`src/app/api/cron/`:**
- Purpose: Scheduled background jobs triggered by Vercel Cron
- Contains: `auto-news/route.ts` (RSS news collection + AI rewrite, daily 21:00 UTC), `auto-press/route.ts` (press release import, daily 09:00 UTC), `backup/route.ts`, `publish/route.ts` (scheduled article publishing)
- Config: Schedules defined in `vercel.json`

**`src/app/cam/`:**
- Purpose: Admin CMS panel with 25+ management pages
- Contains: `dashboard/`, `articles/` (list + new + edit), `headlines/`, `press-import/`, `auto-news/`, `auto-press/`, `mail-press/`, `categories/`, `comments/`, `distribute/`, `seo/`, `rss/`, `ads/`, `popups/`, `newsletter/`, `sns/`, `site-type/`, `settings/`, `about/`, `terms/`, `menus/`, `accounts/`, `analytics/`, `ai-settings/`, `api-keys/`, `logs/`
- Key files: `layout.tsx` (sidebar navigation, auth check, RBAC)

**`src/components/themes/`:**
- Purpose: Switchable theme component sets for public pages
- Contains: `culturepeople/` (5 components), `insightkorea/` (5 components + barrel)
- Each theme: Header, Footer, Landing, ArticlePage, CategoryPage

**`src/components/ui/`:**
- Purpose: Shared UI primitives and ad/widget components
- Contains: shadcn/ui primitives (`button.tsx`, `card.tsx`, `input.tsx`, `dialog.tsx`...), ad components (`AdBanner.tsx`, `AdSenseUnit.tsx`, `CoupangUnit.tsx`, `CoupangAutoAd.tsx`, `FloatingAds.tsx`), widgets (`NewsletterWidget.tsx`, `PopupRenderer.tsx`, `MobileBottomNav.tsx`, `MobileNewsTicker.tsx`, `ScriptUnit.tsx`)

**`src/components/registry/`:**
- Purpose: 100+ pre-built landing page section components (original product - component showcase)
- Contains: One directory per component (e.g., `culturepeople-header-0/`, `agency-hero-with-showcase/`)
- Pattern: Each directory contains a React component file

**`src/lib/`:**
- Purpose: Core business logic, data access, and utilities
- Key pattern: Dual DB layers - `db.ts` (client, uses fetch) and `db-server.ts` (server, direct Supabase/MySQL)
- Key files listed in detail under "Key File Locations" below

**`scripts/`:**
- Purpose: One-off migration, audit, and batch processing scripts
- Contains: 30+ scripts for article migration (`blog-migrate*.mjs`), content audit (`audit-*.mjs`), batch editing (`batch-reedit*.mjs`), image fixing, and metadata management
- Run with: `tsx` (TypeScript) or `node` (ESM .mjs)

**`data/`:**
- Purpose: JSON file-based DB storage (development-only fallback)
- Contains: `articles.json`, `settings.json`
- Generated: Yes (written by `src/lib/file-db.ts`)
- Committed: Yes (development seed data)

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root layout (fonts, third-party scripts, providers)
- `src/app/page.tsx`: Home page (theme switch + article rendering)
- `src/app/article/[id]/page.tsx`: Article detail page
- `src/middleware.ts`: Request middleware (auth, security, routing)

**Configuration:**
- `next.config.ts`: Next.js config (redirects, security headers, CSP, external packages)
- `vercel.json`: Vercel cron schedules (auto-news 21:00 UTC, auto-press 09:00 UTC) + cache headers
- `tsconfig.json`: TypeScript config with `@/` path alias to `src/`
- `eslint.config.mjs`: ESLint configuration
- `postcss.config.mjs`: PostCSS with Tailwind CSS v4
- `components.json`: shadcn/ui component config
- `src/config/site.tsx`: Site metadata config with Zod validation
- `src/config/site-data.json`: Static site data (name, URL, description, stats)

**Core Logic:**
- `src/lib/db-server.ts`: Server-side DB access (Supabase -> MySQL -> File fallback)
- `src/lib/supabase-server-db.ts`: Supabase REST API wrapper (articles CRUD, settings, comments)
- `src/lib/db.ts`: Client-side DB proxy (wraps `/api/db/*` fetch calls)
- `src/lib/cookie-auth.ts`: HMAC token generation/verification, timing-safe comparison
- `src/lib/site-type.ts`: Theme resolver
- `src/lib/fetch-retry.ts`: Retry wrapper for external APIs
- `src/lib/html-utils.ts`: HTML sanitization, tag parsing
- `src/lib/server-upload-image.ts`: Image upload to Supabase Storage with external URL migration

**Authentication:**
- `src/lib/auth.ts`: Client-side auth (checkAuth, login, logout)
- `src/lib/cookie-auth.ts`: Server-side token management
- `src/app/api/auth/login/route.ts`: Login endpoint (sets HttpOnly cookie)
- `src/app/api/auth/me/route.ts`: Session check endpoint

**Types:**
- `src/types/article.ts`: Article, Comment, AdminAccount, AiSettings, AiSkill, AutoNewsSettings, AutoPressSettings, WatermarkSettings
- `src/types/categories.ts`: Category types
- `src/types/metadata.ts`: Component metadata types

**Database Schemas:**
- `supabase-schema.sql`: Supabase table definitions (articles, comments, settings, accounts, logs, etc.)
- `mysql-schema.sql`: MySQL equivalent schema
- `migration.sql`: Incremental migration scripts

## Naming Conventions

**Files:**
- Pages: `page.tsx` (Next.js App Router convention)
- Layouts: `layout.tsx`
- API routes: `route.ts`
- Components: PascalCase (e.g., `AdBanner.tsx`, `RichEditor.tsx`)
- Libraries: kebab-case (e.g., `cookie-auth.ts`, `db-server.ts`, `fetch-retry.ts`)
- Types: kebab-case (e.g., `article.ts`)
- Scripts: kebab-case with `.mjs` or `.ts` extension

**Directories:**
- App routes: kebab-case (e.g., `auto-news`, `mail-press`, `api-keys`)
- Components: kebab-case for directories (e.g., `culturepeople-header-0`)
- Theme directories: kebab-case (e.g., `culturepeople`, `insightkorea`)

## Where to Add New Code

**New Public Page:**
- Create directory in `src/app/[page-name]/page.tsx`
- Add theme-specific rendering if needed (check `getSiteType()`)
- Use `serverGetArticles()` or `serverGetSetting()` from `src/lib/db-server.ts` for data

**New Admin Page:**
- Create directory in `src/app/cam/[page-name]/page.tsx`
- Add menu entry in `src/app/cam/layout.tsx` `MENU_GROUPS` array
- Use `src/lib/db.ts` functions for data access (client-side)
- If reporter access needed, add path to `REPORTER_ALLOWED_PATHS` in `src/middleware.ts`

**New API Endpoint:**
- Create `src/app/api/[category]/[endpoint]/route.ts`
- Export async `GET`, `POST`, `PATCH`, `DELETE` functions
- Add path to middleware matcher in `src/middleware.ts` if auth is required
- Use `serverGet*`/`serverCreate*` functions from `src/lib/db-server.ts`

**New DB Entity:**
- Add TypeScript interface in `src/types/article.ts`
- Add Supabase CRUD functions in `src/lib/supabase-server-db.ts`
- Add server accessor in `src/lib/db-server.ts` (with MySQL/file fallback if needed)
- Add client accessor in `src/lib/db.ts` if admin UI needs it
- Create API route in `src/app/api/db/[entity]/route.ts`
- Add SQL migration in `migration.sql` or `supabase-schema.sql`

**New Theme:**
- Create directory `src/components/themes/[theme-name]/`
- Implement: Header, Footer, Landing, ArticlePage, CategoryPage components
- Export via `index.ts` barrel file
- Add type to `SiteType` union in `src/lib/site-type.ts`
- Add rendering branch in `src/app/page.tsx`, `src/app/article/[id]/page.tsx`, `src/app/category/[slug]/page.tsx`

**New UI Component:**
- Shared primitives: `src/components/ui/[ComponentName].tsx`
- Use shadcn/ui patterns with `cn()` from `src/lib/utils.ts`

**New Utility Function:**
- Server-only: `src/lib/[name].ts` (add `import "server-only"` if needed)
- Client-compatible: `src/lib/[name].ts` (mark with `"use client"` if browser APIs needed)
- Shared types: `src/types/[name].ts`

**New Cron Job:**
- Create `src/app/api/cron/[job-name]/route.ts`
- Add schedule to `vercel.json` `crons` array
- Add `/api/cron/[job-name]` to middleware matcher (already covered by `/api/cron/:path*`)

## Special Directories

- Purpose: Vercel deployment snapshot (contains `.next`, `node_modules`, `public`)
- Generated: Yes
- Committed: Yes (deployment artifact)

**`data/`:**
- Purpose: File-based DB for local development without Supabase/MySQL
- Generated: Written by `src/lib/file-db.ts` at runtime
- Committed: Yes (seed data)

**`scripts/`:**
- Purpose: One-off migration and maintenance scripts (not part of the running app)
- Generated: No (manually authored)
- Committed: Yes

**`.planning/`:**
- Purpose: GSD planning and codebase analysis documents
- Generated: By tooling
- Committed: Selectively

**`src/app/api/_common/`:**
- Purpose: Shared services for the component registry API (v1)
- Note: This is the original product's API layer (landing page component showcase), separate from the news portal APIs
