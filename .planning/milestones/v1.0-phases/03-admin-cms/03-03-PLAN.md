BUG-08: 작성자 select가 authors 목록 로드 전에 빈 값으로 표시되는 문제 수정.
BUG-09: 카테고리 삭제 시 해당 카테고리에 속한 기사가 있는지 확인하지 않는 문제 수정.

수정 방법:

1. 작성자 select 동기화 (BUG-08 — new/page.tsx):
   - edit/page.tsx의 `__unlisted__` 폴백 패턴을 new/page.tsx에도 적용
   - 작성자 select의 value 로직 수정 (line ~438):
     ```tsx
     value={authors.find((a) => a.name === author)?.id || (author ? "__unlisted__" : "")}
     ```
   - select options에 조건부 옵션 추가:
     ```tsx
     {author && !authors.find((a) => a.name === author) && (
       <option value="__unlisted__">{author} (목록에 없음)</option>
     )}
     ```
   - 이렇게 하면 authors 목록이 아직 로드되지 않았거나 로그인 계정이 목록에 없어도 현재 author 이름이 select에 표시됨

2. 카테고리 삭제 경고 (BUG-09 — categories/page.tsx):
   - 카테고리 삭제 핸들러에서 confirm 다이얼로그에 경고 추가
   - 삭제 전에 해당 카테고리 기사 수를 확인하는 것은 추가 API 호출이 필요하므로, 최소 수정으로 confirm 메시지에 경고 문구 추가:
     ```tsx
     if (!confirm(`"${cat.name}" 카테고리를 삭제합니다.\n\n⚠️ 이 카테고리에 속한 기사가 있을 경우, 해당 기사의 카테고리가 유효하지 않게 됩니다.\n\n계속하시겠습니까?`)) return;
     ```
   - 이것은 최소 변경으로, 사용자에게 위험을 알리는 수준. 기사 자동 재분류는 대규모 리팩토링(Out of Scope)에 해당.

핵심: edit/page.tsx에 이미 있는 __unlisted__ 패턴을 new/page.tsx에 적용. 카테고리 삭제는 경고 문구 추가.
