<task type="auto">
  <name>Task 1: 통합 자동 수정 스크립트 작성 및 실행</name>
  <files>scripts/audit-fix-v2.mjs</files>
  <action>
`audit-result-v2.json`을 읽어 유형별로 자동 수정/삭제를 실행하는 통합 스크립트를 작성한다.

기존 `audit-fix.mjs` ~ `audit-fix4.mjs`의 수정 로직을 하나로 통합하되, 신규 유형도 처리한다.

**삭제 처리** (소프트 삭제: `status="삭제"`, `deletedAt=now()`):
- `ENCODING`: 인코딩 깨짐 기사
- `SHORT_BODY`: 본문 50자 미만
- `MISSING_CONTENT`: 제목/본문 누락
- `BLOCKED_KEYWORD`: "전대통령" 포함 기사
- `DUPLICATE_SOURCE_URL`: source_url 중복 (no가 가장 작은 1건만 보존, 나머지 삭제)
- `DUPLICATE_TITLE`: 정규화 제목+같은 날짜 중복 (no가 가장 작은 1건만 보존, 나머지 삭제)

**본문 수정 처리** (body PATCH):
각 기사의 body에 아래 순서로 치환 적용:

1. `RISKY_IMAGE` (AUD-01): 저작권 위험 도메인 이미지의 `<img>` 태그 전체 제거
   - 정규식: `/<img[^>]*src="https?:\/\/[^"]*(?:위험도메인)[^"]*"[^>]*\/?>/gi`
   - 해당 이미지가 `<figure>` 안에 있으면 `<figure>` 전체 제거
   - thumbnail 필드도 확인: 위험 도메인이면 빈 문자열로 PATCH

2. `BASE64_IMG`: `/<img[^>]*src="data:image\/[^"]*"[^>]*\/?>/gi` 제거

3. `TRACKING_PIXEL`: `/<img[^>]*(?:width="1"|height="1")[^>]*\/?>/gi` 제거

4. `OTHER_MEDIA` + `OTHER_REPORTER` + `COPYRIGHT`: 기존 audit-fix.mjs의 치환 로직 그대로 사용
   - 바이라인 줄 전체 제거 (해당 줄만, 본문 전체 삭제 아님)
   - 저작권 문구 줄 제거

5. `NEWSWIRE`: 뉴스와이어 관련 문장/줄 제거
   - `/(?:^|\n)[^\n]*뉴스와이어[^\n]*(?:\n|$)/gi` 등

6. `UI_REMNANT`: UI 잔재 문장 제거

7. `NAMECARD`: 명함/연락처 블록 제거 (해당 단락 전체)

8. `HTML_ENTITY` + `SUMMARY_ENTITY`: 엔티티 디코딩
   - audit-fix.mjs의 `ENTITY_MAP` + audit-fix2.mjs의 `EXTRA_ENTITIES` 통합 사용

9. `EMPTY_TAGS`: 빈 태그 제거 (기존 패턴 그대로)
   - `<p>&nbsp;</p>` 포함

10. `HTML_CLASS`: figure 외부의 class 속성 제거

11. `EXTERNAL_LINK`: `<a href="외부URL">텍스트</a>` → `텍스트` (href만 제거, 텍스트 보존)

12. `AD_PROMO` + `TITLE_MEDIA`: 기존 로직 그대로

13. `WRONG_AUTHOR`: author를 "박영래 기자"로 수정

**구현 규칙:**
- `audit-result-v2.json`에서 문제 기사 목록을 읽어 해당 기사만 처리 (전체 스캔 아님)
- 삭제 대상은 먼저 처리 (수정 불필요한 기사를 건너뛰기 위해)
- 수정 대상은 해당 기사의 현재 body를 Supabase에서 읽어온 후 치환
- 건당 100ms 딜레이 (Supabase API 제한 방지)
- 콘솔에 진행률 표시: `[n/total] 수정 중... (유형: XXX)`
- `--dry-run` 플래그 지원: 실제 PATCH/DELETE 없이 변경 내용만 출력
- 최종 요약: 삭제 N건, 수정 N건, 실패 N건

**주의사항:**
- 수정 시 원본 body 백업은 불필요 (Supabase에 이력 없음, 하지만 dry-run 먼저 실행)
- 중복 삭제 시 보존할 기사(no가 가장 작은)는 절대 삭제하지 않도록 검증
- 이미지 제거 후 본문이 비어버리면 해당 기사도 소프트 삭제
