4건의 RSS 스펙 위반을 수정한다. 최소 변경 원칙 — 기존 구조 유지.

1. **BUG-01 (author 이메일 형식):** 108행의 `<author>${escapeXml(a.author)}</author>`를 RSS 2.0 표준 형식으로 변경:
   ```
   <author>noreply@culturepeople.co.kr (${escapeXml(a.author)})</author>
   ```
   이메일은 `noreply@culturepeople.co.kr` 고정 사용.

2. **BUG-02 (dynamic export 추가):** 파일 최상단(import 후)에 다음을 추가:
   ```typescript
   export const dynamic = "force-dynamic";
   ```
   CDN 캐시는 기존 Cache-Control 헤더로 이미 제어 중이므로 응답 헤더는 변경 불필요.

3. **BUG-03 (enclosure length):** 109행의 `length="0"`은 실제 파일 크기를 알 수 없으므로 enclosure에서 length 속성을 제거하면 안 됨(필수 속성). 대신 추정값이라도 넣기 어려우므로 현재 `length="0"` 유지. 단, 이미지가 없는 경우 enclosure 자체를 생략하는 것은 이미 구현됨 — 변경 불필요.

4. **BUG-16 (비활성화 시 빈 피드):** 44-47행의 비활성화 응답을 RSS 2.0 필수 요소 포함으로 변경:
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <rss version="2.0">
     <channel>
       <title>컬처피플</title>
       <link>https://culturepeople.co.kr</link>
       <description>피드가 비활성화되었습니다.</description>
     </channel>
   </rss>
   ```

BUG-04 (정렬)는 현재 YYYY-MM-DD 형식으로 문자열 비교가 올바르므로 변경 불필요.
