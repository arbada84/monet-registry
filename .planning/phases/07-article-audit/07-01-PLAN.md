기존 `scripts/audit-articles.mjs`를 복사하여 `audit-articles-v2.mjs`로 확장한다.

**기존 14개 유형 유지** (audit-articles.mjs 그대로):
ENCODING, SHORT_BODY, OTHER_MEDIA, COPYRIGHT, HTML_ENTITY, EMPTY_TAGS, OTHER_REPORTER, AD_PROMO, WRONG_AUTHOR, TITLE_MEDIA, HTML_CLASS, SUMMARY_ENTITY, EXTERNAL_LINK, MISSING_CONTENT

**신규 9개 유형 추가** (07-RESEARCH.md 패턴 그대로 사용):

1. `RISKY_IMAGE` (AUD-01): 저작권 위험 이미지 도메인 탐지
   - 07-RESEARCH.md의 `RISKY_DOMAINS` 배열 (48개 도메인) + `isRiskyDomain()` 함수 그대로 사용
   - `body`에서 `src="https://..."` 추출 후 도메인 매칭
   - Supabase/culturepeople 자체 이미지는 제외

2. `DUPLICATE_SOURCE_URL` (AUD-02): source_url 기준 중복
   - 전체 기사 로드 후 `source_url`이 같은 기사 그룹핑
   - 가장 오래된(no가 작은) 1건 제외 나머지를 중복으로 표시

3. `DUPLICATE_TITLE` (AUD-02): 정규화 제목 + 같은 날짜 기준 중복
   - `normalizeTitle(t)` = `t.replace(/\s+/g,"").replace(/[^\p{L}\p{N}]/gu,"").toLowerCase().normalize("NFC")`
   - `created_at` 날짜(YYYY-MM-DD)가 같은 경우만 중복 판정
   - 같은 제목 + 다른 날짜는 탐지만 하고 `DUPLICATE_TITLE_DIFF_DATE`로 별도 플래그 (삭제하지 않음)

4. `NEWSWIRE` (AUD-03): 뉴스와이어 잔재
   - 07-RESEARCH.md의 `NEWSWIRE_PATTERNS` 8개 정규식 사용

5. `UI_REMNANT` (AUD-03): UI 잔재
   - 07-RESEARCH.md의 `UI_REMNANT_PATTERNS` 4개 정규식 사용

6. `NAMECARD` (AUD-04): 명함/연락처 블록
   - 07-RESEARCH.md의 `NAMECARD_PATTERNS` 3개 정규식 사용

7. `BASE64_IMG` (AUD-04): base64 인라인 이미지
   - `/src="data:image\//` 패턴

8. `TRACKING_PIXEL` (AUD-04): 1x1 추적 픽셀
   - `/<img[^>]*(?:width="1"|height="1")[^>]*>/i`

9. `BLOCKED_KEYWORD` (프로젝트 규칙): "전대통령" 키워드 포함 기사
   - `/전대통령/` 제목+본문 검사

**추가 탐지 (수정하지 않고 카운트만):**
- `FORBIDDEN_EXPR`: 금지 표현 (07-RESEARCH.md `FORBIDDEN_EXPRESSIONS` 6개), AI 재편집 후보 플래그용
