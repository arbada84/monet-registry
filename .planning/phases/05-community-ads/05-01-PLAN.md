    - escHtml 함수는 이미 42행에 정의되어 있으므로 그대로 사용.

    **BUG-N02 (MEDIUM — 재구독 토큰 미갱신):** 174행 재구독 로직에서 status만 active로 바꾸는 것에 token도 갱신 추가.
    - 변경 전: `{ ...s, status: "active" as const }`
    - 변경 후: `{ ...s, status: "active" as const, token: crypto.randomUUID() }`
    - 재구독 시에도 웰컴 이메일 발송을 위해, updated 배열에서 해당 구독자를 찾아 sendWelcomeEmail 호출 추가.

    **BUG-N03 (MEDIUM — 인메모리 rate limit):** POST 핸들러의 인메모리 Map 기반 rate limit(126-148행)을 제거하고, redis.ts의 `checkRateLimit` 함수로 교체한다.
    - 파일 상단에 `import { checkRateLimit as redisCheckRateLimit } from "@/lib/redis";` 추가.
    - `subRateLimitMap` 변수와 관련 로직(126-148행) 전체를 삭제.
    - POST 함수 시작부에 다음으로 교체:
      ```
      const ip = request.headers.get("x-forwarded-for")?.split(",")[0]?.trim() || "unknown";
      const allowed = await redisCheckRateLimit(ip, "cp:newsletter:sub:", 5, 3600);
      if (!allowed) {
        return NextResponse.json({ success: false, error: "잠시 후 다시 시도해주세요." }, { status: 429 });
      }
      ```

    **BUG-N04 (LOW — DELETE 인증 누락):** DELETE 핸들러 시작부에 인증 검사 추가.

    - DELETE 함수 try 블록 첫 줄에:
