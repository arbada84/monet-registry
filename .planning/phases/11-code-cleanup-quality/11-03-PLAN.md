1. **eslint.config.mjs** 수정:
   - `"@typescript-eslint/no-explicit-any": "off"` 를 `"@typescript-eslint/no-explicit-any": "warn"` 으로 변경
   - registry 컴포넌트에 대한 별도 규칙이 이미 분리되어 있으므로 그 부분은 건드리지 않음

2. **src/lib/search/index.ts** (13줄) 수정:
   현재: `let db: Orama<any> | null = null;`
   
   Orama의 create() 호출에서 정의하는 스키마를 기반으로 적절한 타입을 사용한다:
   ```typescript
   // 방법 A: Orama의 TypedDocument 또는 InferOramaSchema 활용
   import type { TypedDocument, Orama } from "@orama/orama";
   
   type SearchSchema = {
     id: "string";
     name: "string";
     category: "string";
     searchableText: "string";
     functionalTags: "string[]";
     styleTags: "string[]";
     layoutTags: "string[]";
   };
   
   let db: Orama<{ Schema: SearchSchema }> | null = null;
   ```
   
   방법 A가 타입 호환성 문제를 일으키면:
   ```typescript
   // 방법 B: 최소한의 타입 사용
   // eslint-disable-next-line @typescript-eslint/no-explicit-any -- Orama 내부 타입이 복잡하여 any 사용
   let db: Orama<any> | null = null;
   ```
   
   방법 B는 최후 수단으로, 주석으로 명시적 예외 처리한다.
