# Phase 13 Plan 04: 어드민 UI 주요 플로우 E2E 테스트 구축

## 개요
어드민 페이지의 핵심 기능이 실제 브라우저 환경에서 정상적으로 작동하는지 검증하기 위한 E2E 테스트를 작성합니다.

## 목표
- 로그인부터 기사 작성, 게시, 삭제까지의 전체 플로우 자동 검증.
- 설정 변경 및 저장 기능의 정상 작동 확인.
- `vitest` + `puppeteer` 환경을 활용한 테스트 구축.

## 작업 내용

### Task 1: E2E 테스트 유틸리티 보완
- `e2e/helpers/admin-auth.ts`: 어드민 로그인 과정을 자동화하는 헬퍼 함수 작성.
- `e2e/helpers/article-actions.ts`: 기사 작성, 편집, 삭제 시뮬레이션 함수 작성.

### Task 2: 기사 라이프사이클 테스트 (`e2e/admin/article-lifecycle.test.ts`)
- 신규 기사 작성 및 임시저장 테스트.
- 기사 편집 및 '게시' 상태 전환 테스트.
- 게시된 기사가 실제 공개 페이지에서 접근 가능한지 확인.
- 기사 삭제 후 목록에서 사라지는지 확인.

### Task 3: 설정 관리 테스트 (`e2e/admin/settings.test.ts`)
- 브랜드 설정(사이트명) 변경 및 저장 테스트.
- SMTP 테스트 메일 발송 UI 동작 확인 (실제 발송 여부보다 UI 응답 위주).
- 워터마크 설정 변경 및 유지 확인.

## 검증 계획
- `pnpm test:e2e` 실행 시 모든 테스트 케이스가 실제 서버에서 통과하는지 확인.
- 로컬 개발 환경에서 서버 구동 후 테스트 실행 및 결과 확인.
