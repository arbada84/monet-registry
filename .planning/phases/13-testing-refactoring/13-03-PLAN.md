# Phase 13 Plan 03: `edit/page.tsx` 리팩토링 및 커스텀 훅 추출

## 개요
기사 편집 페이지의 복잡한 상태 관리와 비즈니스 로직을 `useArticleEditor` 훅으로 캡슐화하고, UI 구성을 서브컴포넌트로 분리합니다.

## 목표
- `AdminArticleEditPage`를 300줄 이하로 축소.
- 기사 데이터 로딩, 초안 저장, 제출 로직을 커스텀 훅으로 분리.
- 에디터, 메타데이터 입력, 미리보기 등을 독립된 컴포넌트로 추출.

## 작업 내용

### Task 1: `useArticleEditor` 커스텀 훅 추출
- `src/app/cam/articles/[id]/edit/hooks/useArticleEditor.ts` 생성.
- 기사 필드별 상태, 로딩 상태, `localStorage` 초안 저장 로직 이동.
- `handleSubmit`, `handleReuploadImages`, `handleDelete` 등 액션 핸들러 캡슐화.
- 저장 진행 상태(progress) 및 타이머 로직 포함.

### Task 2: 편집 UI 컴포넌트 분리
- `src/app/cam/articles/[id]/edit/components/ArticleMetadataForm.tsx`: 제목, 카테고리, 태그, 작성자 정보 등.
- `src/app/cam/articles/[id]/edit/components/ArticleThumbnailControl.tsx`: 썸네일 업로드 및 URL 관리.
- `src/app/cam/articles/[id]/edit/components/ArticleActionSidebar.tsx`: 상태 변경, 예약 발행 설정, 저장/삭제 버튼.
- `src/app/cam/articles/[id]/edit/components/ArticlePreviewModals.tsx`: 데스크톱/모바일 미리보기 다이얼로그.

### Task 3: `AdminArticleEditPage` 재구성
- 추출된 커스텀 훅과 컴포넌트들을 사용하여 페이지 로직을 간소화.
- `RichEditor`와 AI 패널 등의 배치 최적화.

## 검증 계획
- 기사 로딩, 임시저장, 게시, 삭제 등의 모든 플로우가 이전과 동일하게 동작하는지 확인.
- 이미지 재이관 및 AI 연동 기능의 정상 작동 확인.
