# Phase 13 Plan 01: 핵심 로직 단위 테스트 구축

## 개요
`src/lib`에 있는 핵심 비즈니스 로직과 크론 작업을 위한 단위 테스트를 작성하여 코드의 안정성을 보장합니다.

## 목표
- `cookie-auth.ts`: 토큰 생성, 검증, 블랙리스트 로직 테스트.
- `db-server.ts`: 기사 생성 시 데이터 정제 로직 테스트.
- `publish/route.ts`: 예약 발행 및 휴지통 삭제 로직 테스트.

## 작업 내용

### Task 1: Vitest 단위 테스트 설정 보완
- `package.json`에 `test:unit` 스크립트 추가 (`vitest run`).
- `vitest.config.ts` (단위 테스트 전용) 작성 또는 기존 설정 활용 검토.

### Task 2: `cookie-auth.test.ts` 작성
- `generateAuthToken`과 `verifyAuthToken`의 왕복 테스트.
- `timingSafeEqual`의 보안적 비교 검증.
- `isTokenBlacklisted`의 Redis 모킹 테스트.

### Task 3: `db-server.test.ts` 작성
- `serverCreateArticle` 내부의 `stripImageSourceCredit` 및 HTML 세정 로직 테스트.
- `getNextArticleNo`의 숫자 생성 규칙 테스트.
- `serverAddViewLog`의 중복 로그 방지 로직 테스트.

### Task 4: `publish-cron.test.ts` 작성
- `runPublish` 함수 내부 로직(기사 상태 변경, 휴지통 삭제 대상 선정) 테스트.
- `db-server` 및 외부 API(`notifyIndexNow`, `notifyNewsletterOnPublish`) 모킹 처리.

## 검증 계획
- `pnpm test:unit` 명령어를 통해 모든 테스트 케이스 통과 확인.
- `coverage` 리포트를 통해 핵심 경로의 커버리지 확인 (선택적).
