마이그레이션 SQL:
```sql
CREATE TABLE IF NOT EXISTS notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL,
  title TEXT NOT NULL,
  message TEXT NOT NULL DEFAULT '',
  metadata JSONB DEFAULT '{}',
  read BOOLEAN NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_notifications_read ON notifications(read);
CREATE INDEX IF NOT EXISTS idx_notifications_created_at ON notifications(created_at DESC);

ALTER TABLE notifications ENABLE ROW LEVEL SECURITY;
CREATE POLICY "notifications_service_all" ON notifications
  FOR ALL USING (true) WITH CHECK (true);
```

async function checkAuth(req: NextRequest): Promise<boolean> {

  return result.valid;

엔드포인트:

**GET** `/api/db/notifications`
- 쿼리파라미터 `unread=1`: 읽지 않은 건수만 반환 `{ count: number }`
- 기본: 최근 50건 반환 `{ notifications: Notification[] }`
- Supabase REST: `GET /rest/v1/notifications?order=created_at.desc&limit=50`
- unread 카운트: `GET /rest/v1/notifications?read=eq.false&select=id` → 배열 길이 반환
  또는 Prefer: count=exact 헤더 사용

**POST** `/api/db/notifications`
- body: `{ type: string, title: string, message?: string, metadata?: object }`
- Supabase REST: `POST /rest/v1/notifications` with body
- 응답: `{ success: true }`

**PATCH** `/api/db/notifications`
- body: `{ ids: string[] }` — 해당 ID들을 읽음 처리
- 또는 `{ all: true }` — 전체 읽음 처리
- all=true: `PATCH /rest/v1/notifications?read=eq.false` body `{ "read": true }`
- ids: 각 ID에 대해 `PATCH /rest/v1/notifications?id=eq.{id}` body `{ "read": true }`

**DELETE** `/api/db/notifications`
- 전체 삭제: `DELETE /rest/v1/notifications?id=not.is.null` (모든 행)
- 응답: `{ success: true }`

모든 엔드포인트에 `checkAuth` 인증 필수.
에러 시 `{ success: false, error: message }` 반환.

Notification 타입 (파일 내 정의):
```typescript
interface Notification {
  id: string;
  type: string;
  title: string;
  message: string;
  metadata: Record<string, unknown>;
  read: boolean;
  created_at: string;
}
```

크론 라우트에서 직접 Supabase REST를 호출하는 대신 이 함수를 사용한다.

```typescript
export async function createNotification(
  type: string,
  title: string,
  message: string = "",
  metadata: Record<string, unknown> = {},
): Promise<void> {
  const url = process.env.NEXT_PUBLIC_SUPABASE_URL;
  const key = process.env.SUPABASE_SERVICE_KEY;
  if (!url || !key) return;
  try {
    await fetch(`${url}/rest/v1/notifications`, {
      method: "POST",
      headers: {
        apikey: key,
        Authorization: `Bearer ${key}`,
        "Content-Type": "application/json",
        Prefer: "return=minimal",
      },
      body: JSON.stringify({ type, title, message, metadata }),
    });
  } catch (e) {
    console.error("[createNotification] failed:", e);
  }
}
```

이 함수는 fire-and-forget 패턴으로, 알림 생성 실패가 크론 작업을 방해하지 않는다.
