<task id="12-02-01">
<title>search_articles RPC 함수 검증 및 갭 분석</title>
<read_first>
- src/lib/supabase-server-db.ts
- .claude/skills/culturepeople-master/SKILL.md (section 2.2 RPC 함수)
</read_first>
<action>
Supabase MCP(`mcp__supabase__execute_sql`)를 사용하여 현재 `search_articles` RPC 함수 정의를 확인한다.

확인 항목:
1. RPC 함수 소스코드 조회:
```sql
SELECT pg_get_functiondef(oid) FROM pg_proc WHERE proname = 'search_articles';
```

2. search_vector 컬럼 존재 확인:
```sql
SELECT column_name, data_type FROM information_schema.columns
WHERE table_name = 'articles' AND column_name = 'search_vector';
```

3. search_vector 업데이트 트리거 확인:
```sql
SELECT trigger_name, event_manipulation, action_statement
FROM information_schema.triggers WHERE event_object_table = 'articles';
```

4. 인덱스 확인:
```sql
SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'articles' AND indexname LIKE '%search%' OR indexname LIKE '%trgm%';
```

**기대 결과:**
- `search_articles` RPC가 `to_tsvector('simple', ...)` 사용 (NOT 'english')
- title에 가중치 A, body에 가중치 B 적용
- `search_vector` 컬럼이 있고, INSERT/UPDATE 트리거가 자동 갱신
- GIN 인덱스 존재

**갭 발견 시:**
- 'english' 파서 사용 중이면 → 'simple'로 변경하는 마이그레이션 SQL 작성 및 적용
- 가중치 미설정이면 → `setweight(to_tsvector('simple', coalesce(title, '')), 'A') || setweight(to_tsvector('simple', coalesce(strip_html(body), '')), 'B')` 패턴으로 업데이트
- 트리거 없으면 → INSERT/UPDATE 트리거 생성

**갭 미발견 시:**
- 현재 구현이 요구사항(D-07~D-10)을 충족함을 확인하고 FEAT-02를 완료로 표시
</action>
<acceptance_criteria>
- search_articles RPC가 존재하며 실행 가능
- RPC 함수가 'simple' 파서를 사용하거나 'simple'로 변경됨
- sbSearchArticles 함수가 supabase-server-db.ts에 존재 (기존 확인)
- 한글 검색어로 search_articles RPC 호출 시 결과 반환됨
</acceptance_criteria>
</task>

<task id="12-02-02">
<title>sbSearchArticles 함수 ilike 폴백 및 pg_trgm 활용 확인</title>
<read_first>
- src/lib/supabase-server-db.ts
- src/app/search/page.tsx
</read_first>
<action>
`src/lib/supabase-server-db.ts`의 `sbSearchArticles` 함수를 검토하여:

1. tsvector RPC 호출 → 결과 있으면 반환
2. RPC 실패 또는 결과 없으면 ilike 폴백
3. 반환 데이터가 검색 결과 페이지에서 필요한 필드를 모두 포함

확인 후 코드 변경 필요 여부 판단:
- ilike 폴백이 있는지 확인
- 함수 시그니처가 `sbSearchArticles(query: string)` 인지 확인
- 반환 타입이 Article[] 인지 확인

**이미 완전한 경우:** 변경 없음, 이 태스크는 검증만 수행
**갭 발견 시:** 구체적 수정 적용 (예: ilike 폴백 추가, 에러 핸들링 보강)

가능한 개선:
- pg_trgm similarity 함수(`similarity()`, `%` 연산자)를 활용한 부분 매칭이 ilike보다 효과적이므로, ilike 폴백을 `similarity()` 기반으로 변경 검토
- 단, 기존 동작이 정상이면 불필요한 변경 자제 (최소 변경 원칙)
</action>
<acceptance_criteria>

- sbSearchArticles has ilike fallback (existing or added)

</acceptance_criteria>
</task>
