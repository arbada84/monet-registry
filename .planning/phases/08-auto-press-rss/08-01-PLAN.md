  <action>
1. `src/lib/newswire-extract.ts` 신규 생성:
   - `isNewswireUrl(url: string): boolean` — `newswire.co.kr/newsRead.php` 포함 여부 확인
   - `extractNewswireArticle(html: string, finalUrl: string)` 함수:
     - 제목: `og:title`에서 ` - 뉴스와이어` 접미사 제거
     - 날짜: `article:published_time` 메타 태그 또는 `div.release-time` 텍스트
     - 작성자: `meta[name="author"]` content
     - 키워드: `meta[name="news_keywords"]` content (쉼표 분리)
     - 본문: `section.article_column` 내부 HTML 추출
     - 본문 정리:
       - `div.release-contact` 영역부터 끝까지 잘라냄 (`indexOf`로 위치 찾아 `slice`)
       - `div.release-source-news`, `div.release-source` 제거 (위 slice로 이미 처리되나 안전 제거)
       - 뉴스와이어 바이라인 제거: `/^[^<]*--\(<a[^>]*>뉴스와이어<\/a>\)--/m` 및 텍스트 버전 `/.*?--\(뉴스와이어\)--/g`
     - 이미지 추출:
       - `data-src` 속성에서 `file.newswire.co.kr` URL 우선 추출 (원본 고해상도)
       - fallback: `img src`에서 추출하되 `/thumb/` 경로는 제외
       - OG 이미지도 포함 (`thumb_480` -> `data` 치환으로 고해상도)
     - `images_column` 복잡 구조를 `<figure><img src="..." alt="..." /></figure>`로 단순화
     - 반환: `{ title, bodyHtml, bodyText, date, images, sourceUrl, author, keywords }`
   - `toPlainText`는 `@/lib/html-extract`에서 import하여 재활용
   - `decodeHtmlEntities`는 `@/lib/html-utils`에서 import

2. `src/app/api/cron/auto-press/route.ts`의 `fetchOriginContent()` (174행 근처) 수정:
   - `import { isNewswireUrl, extractNewswireArticle } from "@/lib/newswire-extract"` 추가
   - HTML fetch 후, `isNewswireUrl(finalUrl)` 또는 `isNewswireUrl(articleUrl)` 확인
   - 뉴스와이어인 경우 `extractNewswireArticle(html, finalUrl)` 호출하여 결과 반환
   - 뉴스와이어가 아닌 경우 기존 범용 추출 로직 유지

주의:
- `toPlainText`를 직접 구현하지 않고 기존 html-extract.ts에서 import (Don't Hand-Roll)
- `decodeHtmlEntities`도 기존 html-utils.ts에서 import
- 정규식 패턴은 RESEARCH.md에서 검증된 것 사용

   - 정부 소스는 이미 `kr_press`, `kr_policy` RSS로 대체됨 (RESEARCH.md 확인)
   - 뉴스와이어 소스는 `nwrss_*` 직접 RSS로 대체됨

3. `src/app/api/cron/auto-press/route.ts`의 `runAutoPress()` 설정 로드 부분:
   - 설정 로드 후 마이그레이션 로직 추가 (DB에 저장된 기존 설정 호환):
     ```typescript
     // DB 설정의 netpro 소스를 RSS로 자동 전환
     const NETPRO_TO_RSS: Record<string, string> = {
       "nw_all": "nwrss_all",
       "nw_economy": "nwrss_econ",
       "nw_culture": "nwrss_cult",
       "gov_policy": "kr_policy",
       "gov_press": "kr_press",
     };
     settings.sources = settings.sources
       .filter(s => !NETPRO_TO_RSS[s.id])  // 넷프로 소스 제거
       .map(s => {
         if (s.fetchType === "netpro" || (!s.fetchType && !s.rssUrl)) {
           s.fetchType = "rss";
         }
         return s;
       });
     ```
   - 대응하는 RSS 소스가 이미 활성화되어 있으면 중복 방지

주의:
- DB에 저장된 `cp-auto-press-settings`에 fetchType: "netpro" 소스가 있을 수 있음 (Pitfall 5)
- 마이그레이션은 런타임에 자동 처리 (DB 직접 수정 불필요)
- `boTable`을 string으로 변경해도 기존 코드의 문자열 비교는 정상 동작
