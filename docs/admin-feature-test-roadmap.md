# 컬처피플 관리자 기능 전수검수 로드맵

작성일: 2026-05-01
범위: `/cam/*` 관리자 화면, `/api/*` 관리자/자동화 API, D1/R2 전환 이후 핵심 데이터 흐름

## 1. 즉시 조치 대상

### P0-001 최고 관리자 계정 저장 장애
- 증상: `/cam/accounts`에서 최고 관리자 계정을 저장하면 서버 오류 또는 저장 실패가 발생할 수 있음.
- 원인: `cp-admin-accounts`는 보호 설정이라 `/api/db/settings` 직접 PUT이 403으로 차단되는데, 계정 관리 화면이 이 경로로 저장하고 있었음.
- 조치: `/api/admin/accounts` 전용 API를 신설해 서버에서 인증, 최고 관리자 권한 확인, 비밀번호 해시, 마지막 활성 최고 관리자 보호, 민감값 제거를 처리.
- 검증: 단위 테스트로 비밀번호 해시 미노출, 기존 해시 보존, 신규 비밀번호 해시, 비최고관리자 저장 차단 확인.

### P0-002 인증/세션/권한 회귀검수
- 로그인 성공, 실패, Redis rate limit, 쿠키 만료, 로그아웃 토큰 무효화 확인.
- 최고 관리자만 가능한 기능과 일반 관리자 가능 기능 분리 확인.
- 모든 `/cam/*` 페이지가 미인증 시 로그인으로 이동하는지 확인.

### P0-003 D1/R2 전환 이후 데이터 읽기/쓰기 검수
- 기사 목록, 기사 상세, 검색, 댓글, 알림, 조회 로그가 D1 read adapter 기준으로 정상 조회되는지 확인.
- 쓰기 경로가 Supabase/D1 모드에 따라 중복 또는 단일 저장되는지 확인.
- Supabase 제한 상태에서도 라이브가 D1/R2로 동작 가능한지 확인.

## 3. 테스트 준비 순서

1. 자동 테스트: `pnpm test:unit`으로 권한, 저장, 마이그레이션 유틸을 먼저 고정.
2. 타입/빌드: `pnpm exec tsc --noEmit`, `pnpm build`로 Next route와 클라이언트 컴파일 확인.
3. 비인증 스모크: 관리자 API가 미로그인에서 401/307로 실패하는지 확인.
4. 인증 스모크: 최고 관리자 계정으로 로그인 후 `/cam/accounts`, `/cam/mail-press`, `/cam/telegram`, `/cam/press-import`, `/cam/articles` 핵심 버튼 실행.
5. 라이브 스모크: 배포 후 `https://culturepeople.co.kr`에서 동일 경로 재확인.

## 4. 남은 고도화 과제

- 관리자 기능별 Playwright 스모크 스크립트에 `ADMIN_SMOKE_USERNAME`, `ADMIN_SMOKE_PASSWORD`를 받아 실제 저장 버튼까지 자동 검증.
- 텔레그램 일일 운영 리포트에 D1/R2 사용량, Supabase 제한 해제 예정일, 실패 자동화 요약 포함.
- 보도자료 자동등록은 “원문 그대로 등록”을 차단하고, 본문 품질 점수와 필수 필드 검사 통과 시에만 등록.
- 기사 자동등록은 저작권 리스크 때문에 기본 비활성 유지, RSS 원문 수집 대신 링크 큐/AI 요약/수동 승인 흐름으로 전환.
