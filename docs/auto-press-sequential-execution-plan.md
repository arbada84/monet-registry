# 보도자료 자동등록 순차 실행 고도화 계획

## 배경

Vercel 서버리스 함수는 실행 시간이 제한되어 있어 보도자료 수집, 이미지 처리, AI 편집, 기사 저장을 한 번의 요청에서 많이 처리하면 50초 안전 마진에 도달할 수 있다. 기존 구조는 서버가 시간 초과 상태를 결과 목록에 표시했지만, 관리자 화면은 이를 "대상 소진"처럼 해석할 수 있어 운영자가 다음 버튼을 직접 눌러야 했다.

## 목표

- 관리자는 수동 실행 버튼을 한 번만 누른다.
- 서버가 50초 안전 마진에 도달하면 현재까지 저장된 결과를 안전하게 반환한다.
- 화면은 반환된 `timedOut`/`continuation` 신호를 보고 자동으로 다음 배치를 이어 실행한다.
- 이미 처리된 원문 URL과 제목은 다음 배치에서 제외한다.
- 중복 저장은 기존 저장 직전 중복 방지 로직으로 한 번 더 막는다.

## 설계

1. 서버 응답 확장
   - `AutoPressRun.timedOut`으로 안전 종료 여부를 명시한다.
   - `AutoPressRun.continuation`에 자동 이어실행 가능 여부, 권장 대기 시간, 안내 문구를 담는다.
   - 기존 시간 초과 안내 행은 유지하되, "다음 실행에서 자동 이어 처리"라는 의미를 분명히 한다.

2. 관리자 화면 순차 실행
   - AI 편집 배치는 5건에서 3건으로 줄여 60초 제한에 걸릴 확률을 낮춘다.
   - 한 배치가 시간 초과되어도 결과가 있으면 2초 대기 후 자동으로 다음 배치를 호출한다.
   - 실제 HTTP 응답이 끊겨 JSON 파싱이 실패해도 중복 방지에 의존해 제한 횟수 내에서 자동 재시도한다.
   - 무한 루프 방지를 위해 최대 배치 수와 연속 무응답 제한을 둔다.

3. 운영자 UX
   - 진행 로그에 "자동 이어 실행"을 표시한다.
   - 시간 초과를 오류가 아닌 "안전 종료 후 이어 실행" 상태로 안내한다.
   - `noAiEdit`은 원문 저장 금지 정책과 충돌하므로 실제 등록 목적이 아니라 검증/스킵 확인 용도임을 표시한다.

## 실패/안전 기준

- 같은 원문 URL/제목은 저장 직전 DB 중복 검사로 재차 차단한다.
- 자동 이어 실행 중 같은 후보가 다시 잡혀도 기존 중복 방지와 `excludeUrls`가 방어한다.
- 연속 3회 응답 파싱 실패 또는 최대 배치 수 초과 시 화면은 멈추고 운영자에게 재실행 안내를 남긴다.
- 서버 측 크론은 기존 단발 실행 구조를 유지한다. 이번 개선은 관리자 화면 수동 실행 UX에 초점을 둔다.

## 구현 반영 기준

- 실등록 AI 편집 배치는 3건 단위로 실행한다.
- 미리보기와 원문 저장 금지 검증은 10건 단위로 실행한다.
- 요청 기사 수는 스킵 수가 아니라 실제 등록 또는 미리보기 성공 수 기준으로 줄인다.
- 서버가 `timedOut` 또는 `continuation.shouldContinue`를 내려주면 화면은 2초 뒤 다음 배치를 자동 호출한다.
- JSON 응답이 끊긴 경우도 서버 저장 직전 중복 방지에 의존해 최대 3회까지 자동 이어 실행한다.
